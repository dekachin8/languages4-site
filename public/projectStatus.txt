## ğŸ“ Key Learnings

**View Transitions:**
- Requires proper event listener reinitialization
- Use `astro:after-swap` event for post-transition logic
- Wrap initialization logic in functions

**Content Collections:**
- Extremely powerful and flexible
- Type-safe validation prevents errors
- Easy to query, filter, and sort
- Optional fields work well with conditional rendering

**Component Architecture:**
- Small, focused components are maintainable
- Reusable across templates
- Consistent design language
- Conditional rendering keeps UI clean

**Pagefind Search:**
- Only works in production builds
- Runs automatically after `npm run build`
- Extremely fast and lightweight

**Image Galleries:**
- Threshold logic prevents empty galleries
- Auto-including card images provides visual richness
- Inline images serve narrative, gallery provides summary
- Border management requires attention to component hierarchy

**Performance Optimization:**
- Static generation = instant page loads
- Tailwind = minimal CSS
- View Transitions = smooth UX
- WebP images = faster loads
